# preview

2_htmls/ディレクトリのHTMLスライドをブラウザでプレビューする。

## 実行コマンド

```bash
npm run preview
```

## 使用例

```bash
# 基本的な使い方（2_htmls/のスライドをプレビュー）
npm run preview

# フィルターを指定（part1で始まるファイルのみ）
npm run preview -- part1

# ディレクトリを指定
npm run preview -- --dir custom_htmls

# ブラウザを開かずにindex.htmlを生成のみ
npm run preview -- --no-open
```

## オプション

| オプション | 説明 | デフォルト |
|-----------|------|-----------|
| `--dir, -d` | HTMLファイルのディレクトリ | `2_htmls` |
| `--filter, -f` | ファイル名フィルター | なし |
| `--no-open` | ブラウザを開かない | 開く |

## 動作

1. 指定ディレクトリ内の `.html` ファイルを取得（`index.html` 以外）
2. ファイル名でソートしてスライド順序を決定
3. `index.html` を生成（既存のものを上書き）
4. ブラウザで開く（`--no-open` 指定時を除く）

## 機能

### ナビゲーション

| 操作 | 説明 |
|-----|------|
| `←` / `→` | 前後のスライドへ移動 |
| `Space` | 次のスライドへ |
| `Home` | 最初のスライドへ |
| `End` | 最後のスライドへ |
| `V` | 検証パネルの表示/非表示 |

### 制約チェック（リアルタイム検証）

スライド読み込み時に自動で以下をチェックし、警告を表示:

| チェック項目 | 説明 |
|-------------|------|
| スライドサイズ | 960x540px であるか |
| テキストラップ | DIV内のテキストが`<p>`等でラップされているか |
| 箇条書き記号 | `<p>`が `-`, `•` 等で始まっていないか |
| スタイル配置 | 背景/ボーダー/シャドウがテキスト要素に直接設定されていないか |
| はみ出し検出 | 要素がスライド境界を超えていないか |
| spanマージン | `<span>`にmarginが設定されていないか |

#### 操作方法

- 左上のステータスアイコンをクリック → エラーパネル表示
- `V` キー → エラーパネルの表示/非表示切り替え
- ブラウザコンソールにも詳細ログを出力
