## 1.1: 高度なプロンプト技法

### Decomposition
- 大きな問題を小さなサブタスクに分解
- 複雑な機能実装を段階的に進めたい時に有効
```
認証機能を分割して:
1. ログイン 2. 登録 3. リセット
```
### Self-Refine
- 自己批評→改善のサイクル
- コード品質を反復的に高めたい時に使う
```
このコードの問題点を指摘して改善版を出して
```
### Tree of Thoughts
- 複数の思考経路を並列探索
- 最適解が不明な設計判断で選択肢を比較
```
3つの異なるアプローチで設計案を比較して
```
### Meta Prompting
- AIにプロンプト自体を改善させる
- 期待通りの結果が得られない時に試す
```
このタスクに最適なプロンプトを提案して
```
